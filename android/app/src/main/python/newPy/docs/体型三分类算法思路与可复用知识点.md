# 体型三分类算法 — 思路详解与可复用知识点

> 算法版本: V2 | 最终准确率: 100% (8/8受试者) | 核心模型: KNN5_dist

---

## 一、问题定义

### 1.1 业务场景

汽车座椅内置 **144 点压力传感器**（靠背72 + 坐垫72），以 ~13Hz 采样率持续输出压力数据。
需要在乘客入座后 **2~3秒内** 自动判断其体型类别，用于后续座椅自适应调节。

### 1.2 分类目标

| 标签 | 英文   | 中文 | 体重范围参考   |
|------|--------|------|----------------|
| 0    | Slim   | 瘦小 | ~45-48kg       |
| 1    | Medium | 中等 | ~60-72kg       |
| 2    | Large  | 高大 | ~90-92kg       |

### 1.3 数据规模

- **8 位受试者**，每人一个 CSV 文件
- 每文件 ~200 帧原始数据，入座筛选后 ~150-200 有效帧
- **总计约 1500 帧有效数据**

### 1.4 核心难点

> **个体差异 > 体型差异**

同类体型的不同个体之间的压力分布差异，往往大于不同体型之间的差异。
这是 V1（62.5%准确率）失败的根本原因 — 逐帧分类时模型实际上在学习"识别个人"而非"识别体型"。

## 三、各环节详细思路

### 3.1 数据加载与预处理

#### 传感器布局（144点）

```
[0-71]:   靠背传感器 (72个)
  [0-5]:    左侧翼矩形 (3×2)     ← 侧面包裹
  [6-11]:   右侧翼矩形 (3×2)     ← 侧面包裹
  [12-71]:  中间大矩阵 (10行×6列) ← 主要承压区

[72-143]: 坐垫传感器 (72个)
  [72-77]:  左侧翼矩形 (3×2)
  [78-83]:  右侧翼矩形 (3×2)
  [84-143]: 中间大矩阵 (10行×6列) ← 主要承压区
```

#### 入座状态检测算法

```python
# 自适应阈值法
threshold = min(坐垫压力) + (max(坐垫压力) - min(坐垫压力)) × 0.3

# 流程:
1. 计算每帧坐垫总压力序列
2. 以自适应阈值找出"连续超阈值"片段
3. 裁剪每段前后各5帧过渡帧（消除入座/离座的不稳定数据）
4. 丢弃长度 < 20帧的短片段
```

**可复用知识点:**
> **自适应阈值 + 前后裁剪** 是传感器时序数据中"有效状态检测"的通用范式。
> 关键参数: 阈值比例(0.3), 裁剪帧数(5), 最小片段长度(20)。
> 比固定阈值更鲁棒，能适应不同体重带来的基线压力差异。

### 3.2 帧级特征工程（122维逐一详解）

对**每一帧**的 144 个传感器点提取 122 维特征。理解这些特征前，先明确输入数据的空间结构:

```
靠背 10×6 大矩阵:                     坐垫 10×6 大矩阵:
      col0  col1  col2  col3  col4  col5       （同结构）
row0  [ .    .     .     .     .     . ]
row1  [ .    .     .     .     .     . ]   row0~2  = 上段 (upper)
row2  [ .    .     .     .     .     . ]   row3~6  = 中段 (middle)
row3  [ .    .     .     .     .     . ]   row7~9  = 下段 (lower)
...                                        col0~2  = 左半
row9  [ .    .     .     .     .     . ]   col3~5  = 右半

另外每侧各有一个 3×2 的侧翼小矩形，包裹在身体两侧。
```

---

#### A. 全局 144 点统计特征（10 维）

把**全部 144 个传感器点当作一个无序数值集合**来统计，不区分靠背/坐垫/位置。

| 特征名 | 公式 | 物理含义 | 体型区分力 |
|--------|------|---------|-----------|
| `full_sum` | `sum(全部144点)` | **总压力** — 所有传感器读数之和 | 最直接的体重指标。90kg 的人约为 45kg 人的 2 倍。**入选Top40** |
| `full_mean` | `mean(全部144点)` | **平均压力** — 总压力/144 | 归一化版本的总压力。**入选Top40** |
| `full_std` | `std(全部144点)` | **压力分散度** — 各点之间的差异大小 | 瘦小者压力集中在少数点 → std 高；高大者覆盖多个点 → std 相对 mean 的比例更低 |
| `full_energy` | `sum(每个点²)` | **总能量** — 压力的平方和 | 平方放大了高压力点的贡献。一个点100+一个0 → 能量10000；两个50 → 能量5000。同时编码**压力大小**和**集中程度**。**入选Top40** |
| `full_rms` | `sqrt(mean(每个点²))` | **均方根值** — 能量的归一化版 | 信号处理中衡量"信号强度"的标准指标，比 mean 对高值更敏感。**入选Top40** |
| `full_max` | `max(全部144点)` | **峰值压力** — 最大的单个传感器读数 | 瘦小者体重集中在少数点 → 峰值可能反而更高；高大者分散 → 峰值不那么极端 |
| `full_p75` | `percentile(75)` | **第75百分位** — 排序后第75%位置的值 | 代表"较高压力区域"的典型值，比 max 更稳定。**入选Top40** |
| `full_p25` | `percentile(25)` | **第25百分位** — 排序后第25%位置的值 | 代表"较低压力区域"的典型值。高大者因覆盖面积大，连低压区域也有较高读数。**入选Top40** |
| `full_iqr` | `p75 - p25` | **四分位距** — 中间50%数据的跨度 | 衡量压力分布的"宽度"，比 std 更抗极端值 |
| `full_nonzero` | `count(值>0) / 144` | **非零比例** — 有多少传感器被"触碰" | 直接反映**身体与座椅的接触面积比**。高大者接触更多传感器 → nonzero 更高 |

**设计直觉:** 这组特征回答最粗粒度的问题 — "这个人坐下来，总体上产生了多大的力？"

---

#### B. 侧翼传感器特征（7 维）

侧翼是座椅两侧的"包裹"部分（靠背侧翼挡住上半身两侧，坐垫侧翼挡住大腿/臀部两侧）。

| 特征名 | 公式 | 物理含义 |
|--------|------|---------|
| `back_wing_sum` | `sum(靠背左翼) + sum(靠背右翼)` | **靠背侧翼总压力** — 上半身两侧对侧翼的挤压。身体越宽越大。**入选Top40** |
| `cush_wing_sum` | `sum(坐垫左翼) + sum(坐垫右翼)` | **坐垫侧翼总压力** — 臀部/大腿两侧对侧翼的挤压。臀部越宽越大 |
| `back_wing_mean` | `(mean(左翼) + mean(右翼)) / 2` | **靠背侧翼平均压力** — sum 的归一化版本。**入选Top40** |
| `cush_wing_mean` | `(mean(左翼) + mean(右翼)) / 2` | **坐垫侧翼平均压力** — sum 的归一化版本 |
| `back_wing_lr_diff` | `|sum(左翼) - sum(右翼)|` | **靠背侧翼左右差** — 上半身是否偏向一侧。值大=坐姿不对称 |
| `cush_wing_lr_diff` | `|sum(左翼) - sum(右翼)|` | **坐垫侧翼左右差** — 臀部是否偏向一侧 |
| `wing_total` | `back_wing_sum + cush_wing_sum` | **全部侧翼总压力** — 身体整体"宽度"指标。**入选Top40** |

**设计直觉:** 侧翼特征专门刻画**身体宽度**。体型分类不仅看体重(上下方向)，还要看身材胖瘦(左右宽度)。一个人可以很重但很瘦长，也可以不重但很宽胖，侧翼能区分这种差异。

#### C. 大矩阵统计特征（靠背39维 + 坐垫39维 = 78维）

以下每个特征都有 `back_`（靠背）和 `cush_`（坐垫）两个版本，分别对 10×6=60 个点计算。

##### C-1. 基础统计（14×2 = 28 维）

| 特征 | 公式 | 物理含义 |
|------|------|---------|
| `{prefix}_mean` | `mean(60个点)` | 该区域的**平均压力强度** |
| `{prefix}_std` | `std(60个点)` | 压力值之间的**离散程度**。均匀受力→std小；集中受力→std大 |
| `{prefix}_sum` | `sum(60个点)` | 该区域的**总压力**。`back_sum` 反映"靠了多少力"，`cush_sum` 反映"坐了多少力" |
| `{prefix}_max` | `max(60个点)` | **峰值压力** — 压力最集中的那个点的读数 |
| `{prefix}_min` | `min(60个点)` | **最低压力** — 通常是身体没碰到的边缘区域，接近0 |
| `{prefix}_median` | 中位数 | **中位压力** — 比 mean 更抗极端值。大部分点低值+少数高值时 median 远小于 mean |
| `{prefix}_p10` | 第10百分位 | **低压区域**典型值。多数是 0 或接近 0 的边缘传感器 |
| `{prefix}_p25` | 第25百分位 | **较低压力区域**的代表值 |
| `{prefix}_p75` | 第75百分位 | **较高压力区域**的代表值 |
| `{prefix}_p90` | 第90百分位 | **高压核心区**的代表值。这些点是身体主要接触区 |
| `{prefix}_iqr` | `p75 - p25` | 压力分布的**中间跨度**，体现压力值的分散范围 |
| `{prefix}_range` | `max - min` | **全距** — 极端值跨度 |
| `{prefix}_cv` | `std / mean` | **变异系数** — 无量纲化的离散度。值大=压力分布非常不均匀 |
| `{prefix}_nonzero` | `count(>0) / 60` | **接触面积比** — 60个点中有多少被身体碰到。高大者覆盖更大 |

**百分位数的直觉理解:**

```
想象60个传感器点按压力从小到大排列:

瘦小者: [0,0,0,0,0,0,...,0,0,50,100,200,300]   → p25≈0,  p75≈低
高大者: [0,5,10,20,50,80,...,150,200,250,300]    → p25≈高, p75≈高

→ p25 和 p75 能很好地区分"身体覆盖面积"差异
```

##### C-2. 能量（2×2 = 4 维）

| 特征 | 公式 | 物理含义 |
|------|------|---------|
| `{prefix}_energy` | `sum(每点²)` | **压力能量**。平方运算放大高压力点的贡献。一个点200→能量40000；两个点各100→能量20000。所以 energy 同时编码了**压力大小**和**集中程度** |
| `{prefix}_rms` | `sqrt(mean(每点²))` | **均方根压力**。energy 的归一化版，消除传感器数量影响 |

##### C-3. 分布形态（3×2 = 6 维）

| 特征 | 公式 | 物理含义 |
|------|------|---------|
| `{prefix}_skewness` | `scipy.stats.skew()` | **偏度** — 分布是否"偏向一边"。正偏=大量低值+少量高值（瘦小者：大部分点没压力，少数点压力大）；负偏=大量高值+少量低值（高大者：大面积有压力） |
| `{prefix}_kurtosis` | `scipy.stats.kurtosis()` | **峰度** — 分布的"尖锐程度"。高峰度=压力集中在少数点（尖分布，瘦小者）；低峰度=压力分散在很多点（平分布，高大者） |
| `{prefix}_entropy` | `-sum(p × log2(p))` | **信息熵** — 分布的"均匀程度"。所有有压力的点值相等→熵最大；压力集中在一个点→熵最小。高大者分布更均匀→熵更大 |

**偏度/峰度/熵的图解:**

```
瘦小者压力分布（直方图）:         高大者压力分布:
  ▌                                 ▌  ▌  ▌  ▌
  ▌                                 ▌  ▌  ▌  ▌  ▌
  ▌  ▌                              ▌  ▌  ▌  ▌  ▌  ▌
──────────                        ──────────────────
正偏度, 高峰度, 低熵              低偏度, 低峰度, 高熵
(集中在少数高值点)                (分散在多个中等值点)
```

##### C-4. 空间分布（9×2 = 18 维）

这组特征开始考虑传感器的**空间位置**，不再是无序统计。

| 特征 | 公式 | 物理含义 |
|------|------|---------|
| `{prefix}_row_max_idx` | `argmax(各行均值)` | **压力最大的行号**（0=最上, 9=最下）。靠背上高大者肩宽→上部行压力大→值更小 |
| `{prefix}_row_spread` | `std(各行均值)` | **行间压力差异** — 从上到下压力是否均匀。值大=身体只接触特定几行 |
| `{prefix}_col_spread` | `std(各列均值)` | **列间压力差异** — 从左到右压力是否均匀。值大=左右某侧压力明显更高 |
| `{prefix}_upper_ratio` | `sum(row0~2) / 总和` | **上段压力占比**（row0~2, 占30%面积）。靠背上段=肩部区域，坐垫上段=大腿根部 |
| `{prefix}_middle_ratio` | `sum(row3~6) / 总和` | **中段压力占比**（row3~6, 占40%面积）。靠背中段=腰背部，坐垫中段=臀部核心 |
| `{prefix}_lower_ratio` | `sum(row7~9) / 总和` | **下段压力占比**（row7~9, 占30%面积）。靠背下段=腰部以下，坐垫下段=大腿前端 |
| `{prefix}_lr_diff` | `|左半-右半| / 总和` | **左右不对称度**（归一化）。≈0=完美对称，值大=偏坐 |
| `{prefix}_lr_ratio` | `左半 / 右半` | **左右压力比** — ≈1.0=对称, >1=偏左, <1=偏右 |

**上中下三段比的身体对应关系:**

```
靠背 (从上到下):                    坐垫 (从上到下):
row0  ┐                            row0  ┐
row1  ├ upper (肩膀)                row1  ├ upper (大腿根/臀后)
row2  ┘                            row2  ┘
row3  ┐                            row3  ┐
row4  ├ middle (腰背)              row4  ├ middle (臀部核心)
row5  │                            row5  │
row6  ┘                            row6  ┘
row7  ┐                            row7  ┐
row8  ├ lower (下腰)               row8  ├ lower (大腿前端)
row9  ┘                            row9  ┘

高大者: upper_ratio 更高（肩宽/臀宽覆盖更多上段面积）
瘦小者: middle_ratio 更高（压力集中在核心接触区）
```

##### C-5. 压力中心 CoP（2×2 = 4 维）

| 特征 | 公式 | 物理含义 |
|------|------|---------|
| `{prefix}_cop_row` | `sum(行号 × 行压力) / 总压力` | **压力中心纵向位置** — 加权平均的"重心行号"。=0全在最上面一行，=9全在最下面。是身体质心在座椅上的纵向投影 |
| `{prefix}_cop_col` | `sum(列号 × 列压力) / 总压力` | **压力中心横向位置** — ≈2.5=正中间，偏离=坐姿倾斜 |

**物理类比:** 跟物理学的"质心"概念完全一样。把每个传感器想象成有质量(压力)的点，CoP 就是这些点的质心。

##### C-6. 接触面积（3×2 = 6 维）

| 特征 | 公式 | 物理含义 |
|------|------|---------|
| `{prefix}_contact_above_mean` | `count(值>均值) / 60` | **超过平均压力的点占比** — 自适应阈值版接触面积 |
| `{prefix}_contact_above_50` | `count(值>50) / 60` | **中等压力以上的点占比** — 固定阈值50。高大者有更多点超过50 |
| `{prefix}_contact_above_100` | `count(值>100) / 60` | **高压力以上的点占比** — 固定阈值100。只有真正被身体压到的区域才超100 |

**多阈值接触面积的直觉 — 压力"等高线":**

```
三个阈值相当于在不同"海拔线"上切割压力地形图:

          ___
         /   \
  ------/-----\------  阈值100 → 只有"山顶"  → contact_above_100
       /       \
  ----/---------\----  阈值50  → "半山腰"以上 → contact_above_50
     /           \
  --/-------------\--  均值    → 自适应切割   → contact_above_mean

高大者: 三个阈值下面积都大 → 一座"宽平的山"
瘦小者: 100阈值面积很小   → 一座"窄尖的山"
```

##### C-7. 峰值集中度（1×2 = 2 维）

| 特征 | 公式 | 物理含义 |
|------|------|---------|
| `{prefix}_peak_ratio` | `max / mean` | **峰均比** — 最大值是平均值的多少倍。比值大=压力高度集中（瘦小者）；比值小=压力较分散（高大者） |

##### C-8. 分块区域（6块×3统计×2区域 = 36 维）

将 10×6 矩阵切成 **2×3=6 个子块**:

```
10×6 矩阵分块:
      col0-1    col2-3    col4-5
row0-4  [0,0]     [0,1]     [0,2]    ← 上半部分
row5-9  [1,0]     [1,1]     [1,2]    ← 下半部分
```

每个子块计算 3 个统计量:

| 特征 | 物理含义 |
|------|---------|
| `{prefix}_block_{i}_{j}_mean` | 该子块的**平均压力** |
| `{prefix}_block_{i}_{j}_sum` | 该子块的**总压力** |
| `{prefix}_block_{i}_{j}_max` | 该子块的**峰值压力** |

**各子块对应的身体部位:**

```
靠背:                               坐垫:
[0,0]=左肩  [0,1]=上背中央 [0,2]=右肩    [0,0]=左臀后 [0,1]=臀部中后 [0,2]=右臀后
[1,0]=左腰  [1,1]=下背中央 [1,2]=右腰    [1,0]=左腿前 [1,1]=大腿中前 [1,2]=右腿前
```

**设计意义:** 全局 mean/sum 是一个数字概括整张矩阵，信息丢失很大。分成 6 块后，保留了**粗粒度的空间模式** — 比如"肩部有压力但腰部没有"这种信息只有分块特征才能捕捉。

##### C-9. 梯度（3×2 = 6 维）

| 特征 | 公式 | 物理含义 |
|------|------|---------|
| `{prefix}_row_grad_mean` | `mean(|diff(各行均值)|)` | **行间压力梯度均值** — 从上到下相邻两行压力变化的平均幅度 |
| `{prefix}_row_grad_max` | `max(|diff(各行均值)|)` | **行间压力梯度最大值** — 压力变化最剧烈的位置。对应身体边缘（身体与空气交界处梯度最大） |
| `{prefix}_col_grad_mean` | `mean(|diff(各列均值)|)` | **列间压力梯度均值** — 从左到右的压力变化程度 |

**梯度的物理直觉:**

```
瘦小者某行压力剖面 (从左到右):
  0  0  20  200  180  0     → 梯度: [0, 20, 180, 20, 180] → 梯度大(陡峭)
                              身体窄，边缘变化剧烈

高大者某行压力剖面:
  30  60  120  150  100  40  → 梯度: [30, 60, 30, 50, 60] → 梯度小(平缓)
                              身体宽，压力过渡平滑
```

##### C-10. 对角线（1×2 = 2 维）

| 特征 | 公式 | 物理含义 |
|------|------|---------|
| `{prefix}_diag_mean` | `mean(mat[i, min(i,5)] for i=0..9)` | **主对角线方向平均压力** — 从左上到右下方向的一条线上的压力均值。10×6非方阵，i>5 时固定在第5列。提供一个与行/列正交的斜向空间采样视角 |

---

#### D. 靠背-坐垫组合特征（14 维）

这组特征**不单看靠背或坐垫**，而是看它们之间的关系。

##### D-1. 全局组合（10 维）

| 特征 | 公式 | 物理含义 |
|------|------|---------|
| `total_pressure` | `back_sum + cush_sum` | **靠背+坐垫总压力**（不含侧翼，与 full_sum 不同）。**入选Top40** |
| `total_energy` | `back_energy + cush_energy` | **总能量**。**入选Top40** |
| `back_cush_ratio` | `back_sum / cush_sum` | **靠坐压力比** — 不同体型"靠"和"坐"的力量分配不同。胖的人更多力量在坐垫（重力通过臀部传导） |
| `back_cush_energy_ratio` | `back_energy / cush_energy` | **靠坐能量比** — 能量版对高值更敏感，捕捉更精细差异 |
| `mean_ratio` | `back_mean / cush_mean` | **靠坐均值比** — 与 back_cush_ratio 高度相关但不完全相同（非零点数可能不同） |
| `std_ratio` | `back_std / cush_std` | **靠坐离散度比** — 上下半身接触模式差异 |
| `cop_row_diff` | `back_cop_row - cush_cop_row` | **靠坐压力中心行差** — 靠背与坐垫重心的纵向偏差，反映前倾/后仰 |
| `entropy_diff` | `back_entropy - cush_entropy` | **靠坐熵差** — 坐垫因直接承重通常更均匀（熵更大） |
| `entropy_ratio` | `back_entropy / cush_entropy` | **靠坐熵比** — 差值的比值版本 |
| `wing_main_ratio` | `wing_total / total_pressure` | **侧翼占比** — 侧翼压力占总压力的比例。**归一化的身体宽度指标**，无论体重大小，身体宽则比值高。**入选Top40** |

##### D-2. 分块比值（6 维）

| 特征 | 公式 | 物理含义 |
|------|------|---------|
| `bc_ratio_{i}_{j}` | `back_block_{i}_{j}_mean / cush_block_{i}_{j}_mean` | **对应位置的靠坐比** — 6 个比值形成一幅"身体前后力量分配地图" |

```
bc_ratio 的空间含义:
  [0,0]: 左上区域 — 肩部 vs 臀后 的前后力分配
  [0,1]: 中上区域 — 上背 vs 臀中
  [0,2]: 右上区域 — 右肩 vs 右臀
  [1,0]: 左下区域 — 左腰 vs 左腿
  [1,1]: 中下区域 — 下背 vs 大腿
  [1,2]: 右下区域 — 右腰 vs 右腿
```

#### E. 时间变异特征（滑动窗口后额外增加 122 维）

以上 122 个特征都是**单帧静态**的。经过滑动窗口聚合后，每个特征都会生成一个 `_tstd` 版本:

| 原特征 | 时变版本 | 含义 |
|--------|---------|------|
| `full_sum` | `full_sum_tstd` | 30帧内 full_sum 的标准差 — 总压力在 2.3 秒内波动了多少 |
| `back_mean` | `back_mean_tstd` | 30帧内 back_mean 的标准差 — 靠背平均压力的时间波动 |
| ... | ... | 每个帧级特征都有对应的时变版本 |

帧级特征回答"这一刻压力是什么样的"；时变特征回答"这段时间内压力**变化了多少**":

```
瘦小者 full_sum 时间序列:         高大者 full_sum 时间序列:
  ╱╲    ╱╲                           ─────────
 ╱  ╲  ╱  ╲  ╱╲                    波动小（坐得稳）
╱    ╲╱    ╲╱  ╲
波动大（接触面积小，微调频繁）

→ full_sum_tstd 大                 → full_sum_tstd 小
```

最终进入模型: **122(均值) + 122(时变) = 244 维** → SelectKBest 选出 **40 维**。

### 3.3 滑动窗口时间聚合（核心突破）

#### 为什么需要窗口聚合？

V1 的帧级分类存在根本问题:

```
V1 的困境:
- 单帧数据噪声大 → 分类不稳定
- 1500帧中信息高度冗余 → 训练效率低
- 个体差异 > 体型差异 → 模型在学"认人"而非"认体型"
```

#### 窗口聚合的两个关键操作

```python
# 参数: window_size=30帧(≈2.3秒), stride=15帧(50%重叠)

for start in range(0, n - window_size + 1, stride):
    window = values[start:start + window_size]  # 取30帧

    mean_feats = np.mean(window, axis=0)   # ① 均值 → 消除噪声
    std_feats  = np.std(window, axis=0)    # ② 标准差 → 捕捉动态模式
```

| 操作 | 产出 | 作用 |
|------|------|------|
| 窗口内均值 | 122维 | 消除帧间噪声，提取稳定的压力"基线" |
| 窗口内标准差 | 122维(`_tstd`后缀) | 捕捉时间变异模式（坐姿微调节的幅度） |
| 合计 | 244维 | 静态特征 + 动态特征 |

#### 为什么标准差也是有用特征？

不同体型的人在坐着时的**微调节模式**不同:
- **瘦小者**: 接触面积小，微小移动导致压力分布剧烈变化 → 标准差大
- **高大者**: 接触面积大，压力分布相对稳定 → 标准差小

这个"时间变异"维度的信息是单帧特征完全无法获得的。

#### 数据膨胀效果

```
原始: 8受试者 × ~190帧/人 ≈ 1500帧
窗口: 8受试者 × ~11窗口/人 ≈ 88窗口

看似"压缩"，实际是:
- 消除了帧间的冗余重复信息
- 每个窗口携带了30帧的聚合信息量
- 特征维度从122 → 244（加入时变特征）
- 信噪比大幅提升
```

**可复用知识点:**
> **滑动窗口聚合** 是处理高频时序传感器数据的通用范式:
> 1. 窗口均值 → 去噪 + 降采样
> 2. 窗口标准差 → 提取动态变化模式
> 3. 重叠滑动（stride < window_size）→ 保留时序连续性 + 增加样本数
>
> 适用场景: IMU传感器、心电信号、振动检测、音频分类等一切"高频采样+需要分类"的场景。
>
> 窗口大小选择: 需覆盖至少一个完整的动作/状态周期。本例中30帧≈2.3秒，
> 足够覆盖一次坐姿微调节。

### 3.4 特征选择（ANOVA F-test）

```python
# 从244维中自动选择最具区分力的40维
selector = SelectKBest(f_classif, k=40)
X_selected = selector.fit_transform(X_scaled, labels)
```

#### 为什么需要特征选择？

- **88个样本，244维特征** → 严重的维数灾难（样本数 < 特征数的3倍）
- KNN 对高维空间尤其敏感（"维度诅咒"使距离度量失效）
- 去除噪声特征可以提升泛化能力

#### 选中的 Top 40 特征分析

```
按类别统计:
  坐垫相关: 20个 (50%) ← 坐垫特征最重要！
  全局特征:  6个 (15%)
  靠背相关:  5个 (12.5%)
  侧翼特征:  3个 (7.5%)
  组合特征:  4个 (10%)
  分块比值:  2个 (5%)
```

**关键发现:** 坐垫特征占了一半，说明**体重差异主要通过坐垫压力体现**。
这符合物理直觉 — 坐垫直接承受体重，靠背只承受倚靠力。

**可复用知识点:**
> **小样本场景下的特征选择至关重要。**
> ANOVA F-test（方差分析）是监督式过滤法的首选:
> - 计算每个特征在不同类别间的方差比
> - F值高 = 该特征在类间差异大、类内差异小
> - 不依赖具体模型，计算快速
>
> 经验法则: 选择的特征数 k ≤ 样本数/2，本例 40 < 88/2=44，满足要求。

### 3.5 标准化处理

```python
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)  # 零均值，单位方差
```

**为什么标准化对 KNN 至关重要?**

KNN 基于距离度量（Minkowski距离），如果特征量纲不同:
- `full_sum` 可能是 0~10000 量级
- `lr_ratio` 可能是 0~2 量级

不标准化时，KNN 实际上只看 `full_sum`，完全忽略小量级特征。

**可复用知识点:**
> 所有基于距离的算法（KNN, SVM, K-Means）**必须**先做特征标准化。
> 树模型（RF, GBDT）不需要，因为它们基于特征排序而非距离。

### 3.6 评估策略: LOSO-CV + 概率软投票

这是整个系统设计中最精妙的部分。

#### 留一受试者交叉验证（LOSO-CV）

```
普通K-Fold:  随机划分 → 同一个人的数据同时出现在训练集和测试集
             → 模型在学"认人"而非"认体型" → 准确率虚高

LOSO-CV:     每次留出一个完整受试者的全部数据作为测试集
             → 训练集中从未见过该受试者
             → 真实模拟"新用户首次使用"的场景
```

```python
logo = LeaveOneGroupOut()
for train_idx, test_idx in logo.split(X, y, groups):
    # train_idx: 其余7人的所有窗口
    # test_idx:  留出1人的所有窗口
    model.fit(X[train_idx], y[train_idx])
    proba = model.predict_proba(X[test_idx])  # 预测留出者的所有窗口
```

#### 概率软投票（Soft Voting）

对留出受试者的所有窗口（约11个），不是简单的多数投票，而是:

```python
# 硬投票（V1方式，不够鲁棒）:
predictions = [model.predict(window) for window in windows]
final = Counter(predictions).most_common(1)[0][0]  # 多数票

# 软投票（V2方式，更精确）:
probabilities = model.predict_proba(all_windows)  # shape: (11, 3)
mean_proba = probabilities.mean(axis=0)            # shape: (3,)
final = argmax(mean_proba)                         # 概率最高的类
confidence = mean_proba[final]                     # 置信度
```

**软投票 vs 硬投票的区别:**

```
假设某受试者有10个窗口，真实类别=Medium:

硬投票场景:
  6个窗口预测 Medium (置信度51%)
  4个窗口预测 Large  (置信度99%)
  → 硬投票: Medium (6/10) ✓ 但不稳定

软投票场景:
  均值概率: Slim=0.05, Medium=0.55, Large=0.40
  → 软投票: Medium (55%) ✓ 综合了所有窗口的概率信息
```

**可复用知识点:**
> **LOSO-CV 是小样本、被试间分类的黄金评估标准。**
> 它回答的是"模型能否泛化到完全未见过的个体？" — 这才是实际部署中的真实场景。
>
> **概率软投票** 比硬投票更鲁棒:
> - 利用了完整的概率分布信息，而非仅取 argmax
> - 高置信度的预测权重自然更大
> - 对个别异常窗口更容忍
>
> 适用场景: 一切"对一个样本有多次观测，需要融合为一个最终决策"的场景。

### 3.7 模型选择: KNN5_dist

#### 候选模型横评

| 模型 | 受试者准确率 | 帧级F1 |
|------|-------------|--------|
| **KNN5_dist** | **100%** | **0.860** |
| KNN3_dist | 100% | 0.845 |
| GradientBoosting | 100% | 0.852 |
| KNN7_dist | 87.5% | 0.833 |
| RandomForest | 75% | 0.715 |
| LogisticRegression | 62.5% | 0.675 |
| SVM_Linear | 62.5% | 0.650 |
| SVM_RBF | 75% | 0.720 |

#### 为什么 KNN 在此场景下表现最好？

1. **小样本友好**: KNN 不需要大量数据来"学习"参数，它直接用训练数据做预测
2. **非参数化**: 不对数据分布做任何假设（如线性可分、高斯分布）
3. **距离加权**: `weights='distance'` 让近邻的投票权重更大，更好地处理类别边界
4. **K=5 的平衡**: 足够平滑噪声（vs K=3），又不至于过度模糊边界（vs K=7）

**可复用知识点:**
> **小样本场景（N < 100）下的模型选择优先级:**
> 1. KNN（距离加权）— 无参数学习，最简单有效
> 2. SVM（线性核）— 小样本下泛化好
> 3. 集成方法（RF/GBDT）— 需要更多数据
> 4. 神经网络 — 数据量不足时极易过拟合
>
> KNN 的 K 值选择: 通常取 sqrt(N_train) 附近的奇数。
> 本例训练集约 77 个窗口（88-11），sqrt(77) ≈ 8.7，取 K=5 或 K=7 均合理。

### 3.8 帧级 vs 受试者级准确率的关系

```
帧级准确率:     87.5% (77/88 窗口)
受试者级准确率: 100%  (8/8 人)
```

这两个指标不矛盾:

```
混淆矩阵 (帧级):
              预测
真实    Slim  Medium  Large
Slim     25      0      0    ← Slim 100%正确
Medium    0     39      3    ← Medium 偶有误判为Large
Large     0      8     13   ← Large 偶有误判为Medium
```

Large 和 Medium 在帧级上有混淆（8个Large窗口被判为Medium），
但软投票聚合后，每个受试者的**平均概率**仍然指向正确类别。

**可复用知识点:**
> 在多次观测融合的场景中，**个体级准确率远高于单次观测准确率**是正常的。
> 只要单次观测准确率 > 50%（二分类）或 > 33%（三分类），
> 多次观测融合后的准确率会随观测次数指数级提升（类似"陪审团定理"）。
>
> 实际意义: 不需要追求完美的帧级分类，只需要保证足够的观测次数和融合策略。

## 四、实时推理流程

训练完成后的部署推理流程:

```
传感器实时数据流 (13Hz)
        │
        ▼
┌─ 入座检测状态机 ─────────────────────┐
│  IDLE → COLLECTING → CLASSIFYING     │
│                                      │
│  1. 检测稳定入座（连续5帧>阈值）       │
│  2. 收集30帧有效数据                  │
│  3. 触发分类                          │
└──────────┬───────────────────────────┘
           ▼
    30帧 → 122个帧特征 × 30帧
           ▼
    窗口聚合: 均值+标准差 → 244维
           ▼
    StandardScaler + SelectKBest → 40维
           ▼
    KNN5_dist.predict_proba() → [P_slim, P_medium, P_large]
           ▼
    argmax → 体型类别 + 置信度
```

**延迟:** 30帧 / 13Hz ≈ **2.3秒**（从入座到出结果）

## 五、可复用知识点总结

### 知识点 1: 传感器矩阵数据的特征工程范式

```
全局统计 → 区域分块 → 空间分布 → 梯度变化 → 跨区域比值
  (粗)                                           (细)
```

**适用于:** 任何矩阵/网格型传感器数据（压力垫、热成像、触摸屏等）

### 知识点 2: 滑动窗口时间聚合

```
高频采样 → 帧级特征 → 滑动窗口 → {均值, 标准差} → 低频高质量样本
```

**核心价值:** 同时完成去噪、降采样、时变特征提取三件事。

**参数选择经验:**
- 窗口大小: 覆盖 1-2 个动作周期
- 步长: 窗口大小的 50%（平衡样本量和独立性）

### 知识点 3: LOSO-CV 评估范式

```
当分类对象是"个体"而非"样本"时:
  ✗ 随机K-Fold → 过拟合，准确率虚高
  ✓ LOSO-CV → 真实反映泛化能力
```

**适用于:** 医疗诊断（按患者分组）、用户行为识别（按用户分组）、
设备故障检测（按设备分组）等。

### 知识点 4: 概率软投票融合

```
当对同一目标有多次观测时:
  ✗ 逐次硬判决 + 多数投票 → 丢失概率信息
  ✓ 逐次概率预测 + 概率均值 + argmax → 充分利用信息
```

### 知识点 5: 小样本分类的技术栈

```
小样本(N < 100) + 高维特征 的最佳实践:
  1. 特征选择降维（SelectKBest / ANOVA）
  2. StandardScaler 标准化
  3. KNN(距离加权) 或 SVM(线性核) 分类
  4. LOSO-CV 或 留一法 评估
```

### 知识点 6: 自适应阈值检测

```
threshold = min + (max - min) × ratio

比固定阈值的优势:
  - 适应不同个体的基线差异
  - 适应传感器漂移
  - 只需要调一个 ratio 参数（0.3）
```

### 知识点 7: V1→V2 的改进思路可复用

```
V1 失败分析:
  → 诊断: 个体差异 > 类别差异
  → 方案: 时间聚合消除个体噪声 + 软投票融合多次观测

通用化:
  当模型在"学习识别个体"而非"学习识别类别"时:
  1. 可视化(t-SNE)确认数据按个体而非类别聚集
  2. 时间/空间聚合减少个体噪声
  3. 添加归一化/比值特征消除个体基线差异
  4. 多观测融合（投票/概率平均）提升鲁棒性
```

## 六、关键参数速查表

| 参数 | 值 | 所在模块 | 说明 |
|------|-----|---------|------|
| 入座阈值比例 | 0.3 | DataLoader | 自适应阈值的比例因子 |
